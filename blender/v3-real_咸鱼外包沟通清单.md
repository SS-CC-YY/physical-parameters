# v3-real 咸鱼外包沟通清单

## 1. 这个项目怎么跟外包解释

可以用比较口语的方式说：

你好，我这边有一组已经做好的 Blender 物理动画，不需要你从零做物理模拟，也不需要你算运动轨迹。现在想找人帮忙把这些“纯净版动画”包装成更真实的 Blender 场景。

简单说，就是原来画面里可能只有球、方块、斜面、地面这些比较干净的物体；你需要帮我加真实背景、真实材质、灯光、相机和道具，让它看起来像是在真实走廊、实验室、体育馆、工作台、仓库坡道这些地方拍出来的。

但有一个很重要的要求：运动轨迹不能改。球什么时候落地、什么时候反弹，方块什么时候撞到挡板，摆什么时候经过底部阻尼区，这些都已经定好了。你主要做视觉包装，不重新做物理。

## 2. 项目背景和用途

这个项目是用来做视频生成/物理理解模型评测的。我们想比较模型在两种画面下的表现：

- Pure 版：背景很干净，物体和物理关系清楚。
- Real 版：物理轨迹一样，但背景、材质和物体外观更接近真实世界。

所以 Real 版不是追求电影级复杂场景，而是追求“真实、清楚、可评测”。场景要像真实拍摄，但主体运动一定要清楚可见。

## 3. 最重要的几句话

找咸鱼外包时，建议一开始就讲清楚这几句：

1. 我已经有 `.blend` 动画文件，你是在这个基础上改真实场景，不是从零做动画。
2. 运动物体的关键帧不能改，轨迹不能改。
3. 尽量不要用 Blender 物理引擎重新模拟运动，位置轨迹最好保留关键帧。
4. 背景要尽量真实，比如实验室、走廊、体育馆、工作台、仓库坡道等。
5. 运动物体也尽量像真实物体，比如苹果、篮球、橡胶球、木箱、塑料滑块、台球等。
6. 不能穿模，不能悬空，不能遮挡关键事件。
7. 最终要交 `.blend` 源文件，并打包贴图、HDR、外部模型资源。

## 4. 先发给外包哪些文件

不要一开始把整个仓库都发过去。建议先只发一个验证样例，最好用 `v3_C`。

### 4.1 验证样例建议发这些

以 `v3_C` 为例：

| 文件 | 是否必须 | 用途 |
| --- | --- | --- |
| `blender/v3/baselinev3_C_block_incline_floor_friction.blend` | 必须 | Pure 版动画源文件，外包主要基于这个改 |
| `blender/v3/baselinev3_C_block_incline_floor_friction_params.json` | 建议 | 说明相机、帧率、物理参数、物体信息 |
| `blender/v3/baselinev3_C_block_incline_floor_friction_trajectory.csv` | 建议 | 说明主体轨迹，可用于检查是否改动轨迹 |
| `blender/v3/exp_C_block_incline_floor_friction.py` | 可选 | 如果对方懂脚本，可作为运动逻辑参考 |
| `blender/v3-real_外包场景与技术需求.md` | 必须 | 正式技术要求和验收标准 |
| Pure 版预览视频或截图 | 强烈建议 | 让对方快速理解动画长什么样 |

### 4.2 批量制作时再发这些

验证样例通过后，再按实验发对应文件：

| 实验 | 必发 `.blend` |
| --- | --- |
| v3_A | `blender/v3/baselinev3_A_ball_projectile_drag_bounce.blend` |
| v3_B | `blender/v3/baselinev3_B_block_spring_damper.blend` |
| v3_C | `blender/v3/baselinev3_C_block_incline_floor_friction.blend` |
| v3_D | `blender/v3/baselinev3_D_ball_em_damping_pendulum.blend` |
| v3_E | `blender/v3/baselinev3_E_ball_rolling_collision.blend` |
| v3_F | `blender/v3/baselinev3_F_two_ball_collision_drag.blend` |
| v3_G | `blender/v3/baselinev3_G_block_spatial_friction.blend` |

每个实验最好同时发：

- 对应 `_params.json`
- 对应 `_trajectory.csv`
- 对应 `exp_*.py`
- 对应 Pure 版预览视频或截图
- `v3-real_外包场景与技术需求.md`

## 5. 要提前谈清楚的条件

### 5.1 工作内容

需要对方确认：

- 是否能基于已有 `.blend` 文件继续制作。
- 是否能保留原关键帧和原轨迹。
- 是否会做真实材质、灯光、场景包装。
- 是否能把运动主体替换成真实外观模型，但不改变轨迹。
- 是否能检查穿模、遮挡、比例问题。

### 5.2 交付内容

提前说清楚交付必须包含：

- `.blend` 源文件。
- 预览图。
- 预览视频。
- 所有贴图、HDR、外部模型资源。
- 简单 README，说明用了哪些资源、主相机是哪一个、是否移动过相机。

### 5.3 版权和资源

需要提前问：

- 使用的模型和贴图是否可商用或可用于论文/项目展示。
- 是否会使用网上下载素材。
- 下载素材是否会一起打包。
- 是否能提供素材来源链接或简单说明。

### 5.4 报价方式

建议不要一开始按全部 7 个实验、几十个场景打包报价。比较稳的方式是：

1. 先报一个 `v3_C` 验证样例。
2. 验证通过后，再按“每个实验/每个场景”报价。
3. 每个场景包含几轮修改要提前说清楚。
4. 复杂室外场景和简单室内场景可以分不同价格。

## 6. 可以直接复制的咸鱼聊天模板

你好，我这边有一组 Blender 动画，想找人帮忙做真实场景包装。动画本身已经做好了，不需要你重新做物理模拟，也不要改运动轨迹。

你的工作主要是：在现有 `.blend` 基础上加真实背景、材质、灯光、相机和道具，让画面看起来更像真实拍摄。比如背景可以做成实验室、走廊、体育馆、工作台、仓库坡道等；运动物体也尽量包装成真实物体，比如篮球、苹果、橡胶球、木箱、塑料滑块、台球这类。

有几个硬要求：

1. 原来的关键帧和轨迹不能改。
2. 尽量不要用 Blender 物理引擎重新模拟主体运动。
3. 不能穿模、悬空，也不能遮挡落地、碰撞、反弹这些关键时刻。
4. 最后需要交 `.blend` 源文件，贴图和外部模型也要一起打包。

我想先做一个验证样例：方块从斜面滑下来，进入平地后撞到挡板再反弹。你可以先看这个 `.blend` 和说明文件，确认能不能做，再给我报一个样例价格。

## 7. 筛选外包时可以问的问题

可以直接问对方：

1. 你能不能基于已有 Blender 文件继续改，而不是从零做？
2. 你能不能保证不改原来的关键帧动画？
3. 如果我要把一个普通球换成篮球或苹果外观，你能不能让它跟随原轨迹？
4. 你会不会打包贴图和外部模型，保证我这边打开不丢资源？
5. 你能不能先给一个低分辨率预览视频让我检查穿模和遮挡？
6. 你做过真实材质、室内/室外场景、灯光渲染吗？

如果对方一直强调“我可以重新模拟得更真实”，要提醒他：这个项目不需要重新模拟，重点是保留原轨迹并提升视觉真实度。

## 8. 验证样例推荐

推荐第一个样例做 `v3_C`：

- 内容：方块从斜面滑下，进入平地，撞到挡板后反弹。
- 原因：这个场景同时有斜面、地面、挡板、碰撞、反弹，很容易检查外包是否会破坏接触边界。
- 推荐真实背景：实验室演示台、木质坡道装置、仓库小坡道、教学演示装置。
- 推荐真实主体：木块、塑料滑块、小货箱。

验收时重点看：

- 方块有没有陷进斜面或地面。
- 斜面和平地交界有没有错位。
- 挡板碰撞时有没有穿模。
- 反弹有没有被遮挡。
- 场景是否真实但不过分杂乱。
