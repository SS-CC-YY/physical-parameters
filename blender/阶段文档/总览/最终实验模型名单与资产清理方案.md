# 最终实验模型名单与资产清理方案

更新时间：2026-05-08

## 一、适合当前 benchmark 的最终模型实验名单

### 1. 开源主实验组

- `Wan 2.2`
- `HunyuanVideo-1.5`
- `CogVideoX1.5`

这三组应作为当前 benchmark 的开源主结果表。原因是：
- 仍可用
- 有官方权重或官方仓库
- 能覆盖 `T2V / I2V / TI2V` 的主流实验形态

### 2. 开源补充组

- `LTX-Video 0.9.8`

这组更适合作为“快速迭代 / I2V 条件生成 / 低时延”补充对照，而不是最核心主表。

### 3. 闭源主实验组

- `Veo 3.1`
- `Runway Gen-4.5`
- `Luma Ray3.14`
- `MiniMax Hailuo 2.3`
- `HappyHorse-1.0`
- `Seedance2.0`

这四组应作为当前 benchmark 的闭源主结果表。它们分别代表 Google、Runway、Luma、MiniMax 在 2026 年 5 月仍可实际使用且未被更新上位模型替代的主力视频模型。

### 4. 不纳入当前正式实验的模型

- `Sora`
- `Wan 2.1`
- `原始 HunyuanVideo`
- `CogVideoX-2B / 5B / 5B-I2V`
- `Runway Gen-4 / Gen-4 Turbo`
- `Luma Ray2`
- `MiniMax Hailuo 02`

原因分别包括：
- 已停用
- 已有官方明确升级版
- 已不再适合作为“最新可用模型”代表

## 二、Top 视角处理结论

- 原始 `CAM_Top` 为严格俯视 `(0, 0, 15)`。
- 该视角在部分实验中会退化为“只有物体尺度变化、缺少有效平面轨迹”的情况，不利于 `L4 Cross-View Consistency` 评测。
- 当前已将 `CAM_Top` 的几何定义改为“保留历史名称，但实际使用沿 `y` 轴镜像的斜视角”，即功能上更接近 `CAM_Main` 的镜像对照视角。
- 因此，历史 `CAM_Top` 渲染产物全部视为旧资产，不建议继续用于正式 benchmark。

## 三、渲染与评测闭环的当前规范

当前闭环应统一为：

1. `build_variant_blends.py --execute`
2. `render_all.py --execute`
3. `extract_seed.py --execute`
4. 模型生成预测视频
5. 轨迹提取为 `trajectory.csv`
6. `batch_benchmark_eval.py`
7. `cross_view_consistency.json` 汇总

其中：
- `blender/variants/` 是 GT 主源
- `blender/renders/` 是 benchmark 视频主源
- `blender/seeds/` 是 continuation / I2V 输入主源
- `code/batch_benchmark_eval.py` 是新的批量评测入口

## 四、资产分类

### 1. 当前主线保留

- `blender/v1`
- `blender/v2`
- `blender/v3`
- `blender/variants`
- `blender/renders`
- `blender/seeds`
- `blender/scenario`
- `blender/prompts`
- `code/v3`
- `code/benchmark_evaluation.py`
- `code/cross_view_consistency.py`
- `code/batch_benchmark_eval.py`
- `code/baseline_v1_t2v_v2_bundle`

### 2. 明确属于旧版/历史分叉/缓存的资产

- `blender/original_v1`
- `blender/__pycache__`
- `code/__pycache__`
- `code/masterv1_tuned_singlefile_scripts_fixed`
- `code/masterv1_tuned_singlefile_scripts_fixed.zip`
- `code/baseline_v1_t2v_bundle`

### 3. 当前建议删除的旧渲染资产

- 所有历史 `CAM_Top` 渲染目录
- 所有历史 `baseline` 渲染目录
- `renders/v3/baselinev3_*` 这类旧版根目录三视角产物

原因是这些目录混入了旧相机定义和旧组织方式，会干扰后续批量跑数与汇总。

## 五、清理原则

- 只删除已经确认不属于当前主线的数据、缓存或脚本分叉。
- 不删除 `scenario`，因为它仍服务于 `V3-Real` 与 I2V anchor。
- 不删除 `seeds`，因为它们已经是 continuation benchmark 的标准输入资产。
- 不删除当前 `variants` 与非过时 `renders`，因为它们仍是 GT 和渲染主集。
