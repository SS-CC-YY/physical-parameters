## 三、摘要

本阶段我们已经基本完成物理参数 benchmark 的主框架搭建，并将整个体系整理为从简单到复杂、从理想到真实的五层结构：`L1` 测最基础的单参数物理敏感性，`L2` 测长时序和多阶段条件下的单参数稳定理解，`L3` 测多规律耦合下的参数分解能力，`L4` 测跨视角一致性，`L5` 测真实视觉环境下的物理稳健性。当前真正落地最完整的是 `L1-L3`，`L4` 已有脚本和部分多视角数据，`L5` 目前完成了方案与外包沟通材料准备。

从实验实现上看，当前工作区已经形成 `17` 个实验单元：`V1` 共 `4` 个实验，`V2` 共 `6` 个实验，`V3` 共 `7` 个实验。对应目录下当前按 canonical 主线维护 `68` 个变体目录。所有主线实验均已接入统一的 GT 导出格式，即每个变体都可以输出 `.blend`、`trajectory.csv` 和 `params.json`，为后续轨迹拟合和参数反推提供了统一接口。

`V1` 对应 `Level 1: Minimal Parameter Probes`，目标是用最小、最纯净、最易解析的场景测试模型是否具备基础物理参数敏感性。`V1-A` 是自由落体实验，隐藏参数为重力加速度 `g`，评估方向是看模型生成结果中物体竖直轨迹能否支持稳定反推 `g`，本质上测试最基础的重力识别能力。`V1-B` 是竖直弹跳实验，隐藏参数为恢复系数 `e`，评估方向是看模型能否在单次接触事件中保留正确的速度衰减和反弹高度关系。`V1-C` 是水平减速滑行实验，隐藏参数为摩擦系数 `μ`，评估方向是看模型是否能从减速轨迹中恢复摩擦强弱，而不是只生成一个表面上“在滑动”的视频。`V1-D` 是阻尼摆实验，隐藏参数为阻尼系数 `γ`，评估方向是看模型是否能够反映振幅包络的衰减趋势，并支持对 `γ` 的稳定拟合。如果模型在这一层都不能稳定反推出参数，就说明其尚不具备基础的物理参数感知能力。

`V2` 对应 `Level 2: Extended Single-Parameter Dynamics`，目标是在仍保持单隐藏参数的前提下，显著提升时序结构复杂度，以测试模型能否在更长因果链上维持同一物理解释。`V2-A` 是抛射加落地反弹实验，隐藏参数仍为 `g`，评估方向从静态重力识别升级为跨阶段的重力一致性识别。`V2-B` 是连续多次弹跳实验，隐藏参数为 `e`，评估方向是看模型能否在多次接触事件中维持一致的反弹规律。`V2-C` 是斜面下滑加地面滑行实验，隐藏参数为 `μ`，评估方向是看模型能否在不同运动阶段上保持对同一摩擦参数的一致表达。`V2-D` 是长时序阻尼摆，隐藏参数为 `γ`，评估方向是观察模型是否能够在更长时间范围内维持正确的周期与衰减包络。除此之外，当前实现还扩展了 `V2-E` 倾斜地面连续弹跳和 `V2-F` 受迫阻尼摆两个实验，前者进一步强化碰撞恢复参数在复杂边界条件下的稳定性测试，后者则把阻尼识别扩展到受迫振动情形。这一层的核心判断标准是：模型是否只会拟合局部短片段，还是已经能够在长时序和多阶段因果链中保持对单一物理参数的稳定理解。

`V3` 对应 `Level 3: Coupled Multi-Physics Probes`，这是当前 benchmark 的核心层。这里不再只测试单参数，而是要求多个物理规律和多个隐藏参数共同作用，观察模型是否真正学会了物理参数分解与规则组合。`V3-A` 是抛射、空气阻力与反弹耦合实验，隐藏参数为 `g, c, e`，评估方向是测试模型能否同时分解重力、阻力和碰撞恢复三种因素。`V3-B` 是弹簧-阻尼振动实验，隐藏参数为 `ω0, γ`，评估方向是测试模型是否能够同时区分振荡频率和阻尼衰减。`V3-C` 是斜面下滑、地面滑行与挡板碰撞实验，隐藏参数为 `g, μ, e`，评估方向是测试模型对摩擦、重力和碰撞恢复三者耦合的分解能力。`V3-D` 是局部阻尼摆实验，隐藏参数为 `g, γ_zone`，评估方向是看模型能否识别空间非均匀物理场，而不是默认全局参数恒定。`V3-E` 是滚动球下坡、地面滚动与碰墙实验，隐藏参数为 `μ_r, e`，评估方向是测试模型是否具备滚动约束和滚动阻力理解，而非仅仅把运动当作平移轨迹处理。`V3-F` 是双球碰撞与碰后阻力衰减实验，隐藏参数为 `e, c`，评估方向是测试模型对多物体交互、碰撞时序和碰后双轨迹外推的能力。`V3-G` 是空间非均匀摩擦地面实验，隐藏参数为 `μ1, μ2`，评估方向是测试模型能否识别局部规则变化，而不是强行用单一全局参数解释整段运动。总体而言，`V3` 的目标是回答一个更核心的问题：模型到底是在记忆某种常见运动模板，还是已经能够对耦合物理因素进行可解释的分解。

在评测方法上，当前项目已经完成从“直接问模型参数是多少”向“视频续写 -> 轨迹提取 -> 参数反推”的范式切换。也就是说，实际评估时并不是让模型直接输出 `g` 或 `μ`，而是先给它 seed 视频或 seed 图像以及统一提示词，让模型续写完整视频，再从生成结果中提取物体轨迹，并用对应物理公式或分段模型去拟合隐式参数。这个流程的优势在于，它更符合视频生成模型的真实使用方式，也更适合在论文中构造客观、可重复的物理评测标准。当前 `blender/prompts/` 下的 `17` 份提示词都已经统一重写为这一范式，代码侧也已经具备轨迹误差、速度误差、DTW、事件时刻误差、参数拟合误差以及跨视角一致性聚合等基础能力。

从渲染资产来看，当前已经不是只有单视角样例，而是同时具备主线单视角数据和部分多视角基础。`V1` 目前以 `CAM_Side` 为主，适合作为最小基础 benchmark；`V2` 中 `v2_B / v2_C / v2_D / v2_E` 已存在比较完整的三视角目录；`V3` 则同时保留了历史 baseline 三视角渲染，以及新版 variant 主线中较完整的 `CAM_Side` 数据。因此，`L4 Cross-View Consistency` 已经具备启动条件，但新版 variant 目录下的 `CAM_Main` 与 `CAM_Top` 仍需系统补齐，尤其是建议优先补 `V3-A / V3-C / V3-D / V3-F` 这几类最有代表性的实验。

下一阶段的工作重点主要有四项。第一，继续做文档、代码和产物目录的最终冻结对齐，形成一份正式的 canonical 主表，避免后续论文写作和跑数阶段出现口径漂移。第二，补齐 `V3` 关键实验在 `CAM_Main` 和 `CAM_Top` 下的新版 variant 渲染，形成真正可支撑 `L4` 章节的三视角对照集。第三，继续完善“生成视频 -> 自动追踪 -> 参数反推 -> 结果汇总”的批量评测闭环，使其能够稳定输出按模型、按实验、按视角汇总的结果表。第四，从 `V3` 中选择代表性实验启动 `V3-Real` 小规模 pilot，优先验证真实场景下的目标可见性、可追踪性和物理事件可辨识度，为 `L5 Realistic Visual Robustness` 做准备。

总体来看，当前工作已经完成 benchmark 的主体搭建，正在从“能生成 benchmark 数据和评测脚本”的原型阶段，向“能批量评估模型并支撑论文写作”的正式 benchmark 阶段过渡。如果接下来顺利补齐跨视角主线、批量评测闭环和真实场景 pilot，这套 benchmark 就能够较完整地回答视频生成模型是否真正具备物理参数敏感性、长时序一致性、多规律耦合理解能力以及跨视角稳健性的核心问题。
