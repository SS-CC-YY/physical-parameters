# V 系列实验重构方案

## 一、总体目标

当前 `V1 / V2 / V3` 的问题不是实验数量不足，而是层级边界不够清晰：有些实验同时改了背景、规律、物体、参数数量，导致版本之间出现重复和混乱。

因此，新的设计目标是把三代实验重新组织成一条清晰的递进主线：

1. `V1`：单物体、单运动、单隐藏物理参数、理想环境
2. `V2`：单物体、长时序或多阶段运动、仍然单隐藏物理参数、理想环境
3. `V3-Pure`：单物体或极少量交互、多规律耦合、多个隐藏物理参数、理想环境
4. `V3-Real`：在 `V3-Pure` 相同物理核上增加真实背景、真实材质、真实镜头和遮挡

这样每一代的升级点就明确了：

- `V1` 测最基础的单参数识别能力
- `V2` 测模型在更长因果链中的单参数稳定理解能力
- `V3-Pure` 测模型对多参数、多规律耦合的分解能力
- `V3-Real` 测模型在视觉复杂度提升后的物理稳健性

## 二、各版本的定位

### V1：Minimal Physics Probes

核心特点：

- 单个物体
- 单段运动
- 只保留 1 个隐藏参数
- 尽量使用最简单解析公式

推荐实验：

1. 自由落体：隐藏 `g`
2. 竖直弹跳：隐藏 `e`
3. 水平减速滑行：隐藏 `mu`
4. 阻尼摆：隐藏 `gamma`

### V2：Extended Single-Parameter Dynamics

核心特点：

- 仍然是理想环境
- 仍然只保留 1 个隐藏参数
- 但运动变成长时序或多阶段
- 允许边界交互，但不引入多参数耦合

推荐实验：

1. 抛射 + 落地 + 反弹：隐藏 `g`
2. 多次连续弹跳：隐藏 `e`
3. 斜面下滑 + 地面继续滑行：隐藏 `mu`
4. 长时序阻尼摆：隐藏 `gamma`

### V3-Pure：Coupled Multi-Physics Probes

核心特点：

- 仍然是纯净背景
- 引入 2 个以上可拟合隐藏参数
- 明确多规律耦合
- 仍然要求轨迹可由解析公式或近解析分段公式控制

推荐实验：

1. 抛射 + 空气阻力 + 落地反弹：隐藏 `g, c, e`
2. 弹簧-阻尼振动：隐藏 `omega0, gamma`
3. 斜面下滑 + 地面滑行：隐藏 `g, mu`
4. 电磁阻尼摆：隐藏 `g, gamma`

### V3-Real：Realistic Visual Physics Probes

核心特点：

- 不改 `V3-Pure` 的物理核心
- 只改变视觉域：背景、材质、遮挡、光照、相机
- 用于区分“物理理解失败”和“视觉感知失败”

推荐做法：

- 每个 `V3-Pure` 实验派生出多个真实场景版本
- 物理脚本保持一致，只替换环境搭建脚本

## 三、版本之间的明确区别

### V1 与 V2 的区别

`V1` 和 `V2` 都是单参数，但：

- `V1` 是最小 probe
- `V2` 是多阶段或长时序 probe

也就是说，`V2` 的升级点不是“参数更多”，而是“时序结构更复杂，但参数仍然单一”。

### V2 与 V3 的区别

`V2` 和 `V3` 都可以是理想环境，但：

- `V2` 只允许 1 个隐藏参数
- `V3` 明确要求多个隐藏参数共同作用

所以 `V3` 的升级点是“多规律耦合”和“多参数分解能力”。

## 四、推荐的最终结构

### V1

- A: Ball free fall -> `g`
- B: Ball bounce -> `e`
- C: Ball slide -> `mu`
- D: Ball pendulum -> `gamma`

### V2

- A: Ball projectile + bounce -> `g`
- B: Ball repeated bounce -> `e`
- C: Block incline + floor slide -> `mu`
- D: Ball long damped pendulum -> `gamma`

### V3-Pure

- A: Ball projectile + drag + bounce -> `g, c, e`
- B: Block spring-damper oscillation -> `omega0, gamma`
- C: Block incline-to-floor slide -> `g, mu`
- D: Ball electromagnetic damping pendulum -> `g, gamma`

### V3-Real

- 在 `V3-Pure` 的基础上派生真实背景版本
- 保持相同轨迹和相同物理参数定义

## 五、实验设计原则

后续不论增加什么实验，都建议先检查以下 4 条：

1. 能否清楚写出“隐藏参数是什么”
2. 能否清楚写出“用哪条轨迹和哪条公式反推该参数”
3. 该实验属于“单参数长时序”还是“多参数耦合”
4. 该实验的新增复杂度来自“物理”还是“视觉”

如果一个实验同时改动太多维度，或者无法从轨迹中稳定反推参数，就不应该进入主 benchmark。
